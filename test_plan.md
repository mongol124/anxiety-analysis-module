# Test Plan — Anxiety Helper

## 1. Цели тестирования
Цель тестирования — проверить корректность работы функционала приложения Anxiety Helper, убедиться в стабильности, удобстве интерфейса и выявить ошибки.  
Проводится два типа тестирования:
- Функциональное (проверка работы ключевых модулей)
- Пользовательское (оценка UX и логики использования)

## 2. Объекты тестирования
- Модуль диагностики тревожности (Спилбергер, GAD-7)
- Модуль рекомендаций
- Модуль упражнений (дыхательные техники, CBT и др.)
- Дневник настроения
- Аналитика и графики
- Система локального хранения данных

## 3. Среда тестирования
- ОС: Windows 10, macOS Ventura  
- JDK: 17  
- JavaFX: 17  
- Сборка: Maven 3.6  
- DB: SQLite + шифрование  

## 4. Инструменты
- JUnit 5 — модульные тесты  
- JavaFX Test Harness — GUI-тесты  
- Google Forms — сбор обратной связи  
- GitHub Issues — фиксация ошибок  

## 5. Методика тестирования
- Функциональное тестирование выполняется вручную по заранее созданным тест-кейсам.
- Пользовательское тестирование проводится на одногруппниках (5 человек).
- Результаты фиксируются в отчётах: test_report, feedback_summary.

## 6. Критерии завершения тестирования
Тестирование считается завершённым, если:
- Все критические ошибки устранены
- 90% тест-кейсов успешно пройдены
- Есть собранная пользовательская обратная связь
- Сформирована документация в /testing
